# 1. Purpose

The objective is not to recreate FireRed.

The objective is to understand its recurring design decisions well enough to derive reusable principles.

## Rule ATLAS-001

Record observations before drawing conclusions.

Severity: Critical

---

# 2. Measurement Principles

Every measurement should be:

- Repeatable
- Objective
- Comparable
- Traceable to a screen or map

Avoid qualitative statements such as:

- "Feels good"
- "Looks balanced"

Instead record measurable evidence.

---

# 3. Standard Survey Template

For every map:

- Region name
- Map type
- Primary purpose
- Dominant biome
- Connected regions
- Screens
- Primary landmark
- Secondary landmarks
- Pattern chain
- Rule references

---

# 4. Town Analysis Template

Collect:

- Building count
- Landmark count
- Public/private ratio
- Average building spacing
- Average path width
- Open-space percentage
- District count
- Entrance count
- Landmark visibility

Questions:

- Where does the eye move first?
- What defines the town's identity?
- Which rules explain the layout?

---

# 5. Route Analysis Template

Measure:

- Length
- Branch count
- Dead ends
- Optional paths
- Trainer spacing
- Item spacing
- Tall-grass coverage
- Compression/release sequence
- Chokepoints
- Major reveals

Represent routes as gameplay beats instead of only geometry.

---

# 6. Screen Analysis Template

For each camera screen:

- Primary focal point
- Visual weight map
- Dominant silhouette
- Traversable area ratio
- Negative-space ratio
- Landmark visibility
- Grid exposure estimate
- Density level
- Emotional intent

---

# 7. Quantitative Metrics

Recommended measurements:

- Average route width
- Median clearing size
- Tree cluster size
- Forest edge depth
- Water-edge complexity
- Cliff continuity
- Landmark visibility distance
- Building setback
- Screen density
- Decoration density

Do not assume ideal values. Record distributions first.

---

# 8. Comparative Analysis

Compare maps by function rather than appearance.

Example:

Tutorial Route
vs.
Late-Game Route

Focus on:

- rhythm
- complexity
- branching
- openness
- landmark frequency

---

# 9. AI Extraction Workflow

1. Capture reference.
2. Separate layers.
3. Measure.
4. Classify patterns.
5. Link rules.
6. Store metrics.
7. Compare against atlas averages.

Output:

```markdown
Map Summary
Measurements
Pattern Chain
Rule References
Lessons
Open Questions
```

# 10. Future Atlas Structure

Each FireRed area should eventually receive:

- Annotated overview
- Screen-by-screen breakdown
- Pattern map
- Metric tables
- Rule references
- Lessons transferable to new projects

This atlas is intended to become the empirical foundation supporting the earlier volumes.

---

# Final Principle

The strongest design systems are grounded in observation.

The atlas turns admired maps into measurable data that can inform future designs without copying them.
