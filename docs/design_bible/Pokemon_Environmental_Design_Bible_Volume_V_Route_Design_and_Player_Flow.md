# 1. Route Philosophy

A route is not a hallway.

A successful route continuously answers three questions:

- Where can I go?
- What looks interesting?
- What happens if I leave the main path?

## Rule ROUTE-001 — Continuous Curiosity

Every 10–20 seconds of travel, present at least one of:
- a landmark,
- a decision,
- a reward,
- a trainer,
- a biome shift,
- or a vista.

Severity: High

---

# 2. Route Taxonomy

Each route should declare a primary identity.

Types include:

- Connector
- Tutorial
- Exploration
- Challenge
- Puzzle
- Scenic
- Climactic
- Shortcut
- Secret

## Rule ROUTE-010

A route should have one dominant identity and at most two supporting identities.

---

# 3. Player Flow

Movement should alternate between compression and release.

Compression:
- narrow passages
- forest corridors
- bridges
- cliffs

Release:
- clearings
- overlooks
- plazas
- meadows

## Rule FLOW-001

Avoid maintaining the same path width for the entire route.

## Rule FLOW-002

Every major compression should eventually be rewarded with a release.

---

# 4. Visual Guidance

Guide without arrows.

Methods:
- path curvature
- landmark framing
- terrain contrast
- NPC orientation
- lighting
- negative space

## Rule GUIDE-001

The intended direction should be visible without requiring UI.

Severity: Critical

---

# 5. Encounter Pacing

Combat should have rhythm.

Example:

Safe → Trainer → Rest → Tall Grass → Reward → Safe

Avoid:
Combat → Combat → Combat → Combat

## Rule PACE-001

Alternate cognitive load.

## Rule PACE-002

Introduce recovery after difficult encounters.

---

# 6. Optional Exploration

Every route should contain optional value.

Possible rewards:
- item
- shortcut
- lore
- vista
- NPC
- collectible
- hidden passage

## Rule EXPLORE-001

Optional content should never block critical progression.

## Rule EXPLORE-002

Visible rewards encourage exploration better than invisible ones alone.

---

# 7. Reward Placement

Rewards should create decisions.

Poor:
Item directly on main path.

Better:
Visible item requiring a small detour.

Best:
Visible reward plus unknown opportunity beyond it.

## Rule REWARD-001

Reward effort proportionally.

---

# 8. Chokepoints

Chokepoints create memorable transitions.

Examples:
- cave entrance
- bridge
- gate
- canyon
- stair
- forest opening

## Rule CHOKE-001

Every chokepoint should communicate a meaningful spatial change.

---

# 9. Sightlines

Players plan ahead using vision.

Reveal:
- buildings
- mountains
- towers
- bridges
- exits

Hide:
- secrets
- surprises
- encounters

## Rule SIGHT-001

Show destinations before players reach them whenever practical.

---

# 10. Route Rhythm

Think in beats.

Example:

Start
→ Open field
→ Trainer
→ Forest
→ Item
→ Bridge
→ Tall grass
→ Overlook
→ Town

## Rule RHYTHM-001

No two consecutive screens should provide identical experiences.

## Rule RHYTHM-002

Alternate movement, combat, discovery, and visual rest.

---

# 11. Failure Catalog

RF-001 Corridor Syndrome
: Route behaves like a straight hallway.

RF-002 Constant Combat
: No recovery or exploration.

RF-003 Landmark Starvation
: Nothing memorable across the route.

RF-004 Hidden Main Path
: Navigation depends on trial and error.

RF-005 Reward Without Decision
: Valuable items require no exploration.

RF-006 Identical Screens
: Every screen repeats the same composition.

---

# 12. AI Route Audit

Evaluate:

- Primary route visible?
- Optional exploration meaningful?
- Density changes over time?
- Landmarks memorable?
- Chokepoints intentional?
- Encounter pacing varied?
- Player rewarded for curiosity?
- Visual rhythm preserved?

Output format:

```markdown
# Route Audit

Critical
High
Medium

Recommended Revision Order

Do NOT:
- increase art fidelity
- add decorative clutter
- replace sprites

Only improve player flow and spatial relationships.
```

---

# Final Principle

A player should remember **the journey** between towns, not just the towns themselves.

Routes succeed when movement feels authored rather than merely connected.
